Cloud Commerce API

Nowoczesny interfejs REST do zarządzania sklepem internetowym. Zbudowany na FastAPI z bezpieczną autoryzacją JWT i szyfrowaniem haseł Bcrypt.

Explore Docs GitHub

O aplikacji

Cloud Commerce API to system REST API zbudowany w oparciu o FastAPI i SQLite. Aplikacja umożliwia pełne zarządzanie sklepem internetowym — od rejestracji użytkowników, przez zarządzanie produktami, po tworzenie i śledzenie zamówień.

Architektura

System wykorzystuje nowoczesne praktyki bezpieczeństwa i architekturę chmurową:

  • ✓ Tokeny JWT (JSON Web Tokens) dla autoryzacji
  • ✓ Haszowanie haseł algorytmem Bcrypt
  • ✓ Walidacja danych wejściowych (Pydantic)
  • ✓ Rola-bazowana kontrola dostępu (RBAC)
  • ✓ Baza danych relacyjna SQLAlchemy ORM

Przepływ autoryzacji

1. Rejestracja użytkownika

POST /register { "username": "jan", "password": "moje_haslo_123" }

2. Logowanie i uzyskanie tokena

POST /login { "username": "jan", "password": "moje_haslo_123" } Odpowiedź: { "access_token": "eyJhbGciOiJIUzI1NiIsInR5...", "token_type": "bearer" }

3. Użycie tokena w żądaniach

POST /orders Headers: Authorization: "Bearer eyJhbGciOiJIUzI1NiIsInR5..." { "product_id": 5, "quantity": 2 }

4. Walidacja danych (422 Bad Request)

Jeśli wysłesz niewalidne dane, API zwraca 422 z detalami błędu:

{ "detail": "Błąd walidacji danych", "errors": [ { "pole": "body → quantity", "problem": "ensure this value is greater than 0" } ] }
