E-commerce REST API

Projekt zaliczeniowy z przedmiotu: Wprowadzenie do przetwarzania w chmurze

Opis projektu

Aplikacja udostępnia interfejs programistyczny (API) do zarządzania sklepem internetowym. System został zbudowany z wykorzystaniem frameworka FastAPI oraz bazy danych SQLite. Całość została przygotowana zgodnie z architekturą chmurową, używaną na zajęciach.

Opis działania systemu

Proces korzystania z API przebiega w następujących etapach:

  1. Rejestracja: Użytkownik tworzy konto. Hasło jest automatycznie haszowane (bcrypt) przed zapisaniem w bazie SQLite.
  2. Uwierzytelnianie: Po podaniu poprawnych danych logowania, serwer generuje i zwraca podpisany kluczem prywatnym token JWT.
  3. Autoryzacja: Token ten musi być dołączony do nagłówka (Header) przy próbie złożenia zamówienia. Serwer weryfikuje podpis tokena, zapewniając bezpieczeństwo danych.
  4. Zarządzanie danymi: Każdy użytkownik może przeglądać produkty, ale tylko zalogowani użytkownicy z ważnym tokenem mogą tworzyć zamówienia, które są powiązane z ich identyfikatorem w bazie danych.

Bezpieczeństwo

  • Autoryzacja za pomocą tokenów JWT (JSON Web Tokens).
  • Hasła użytkowników są szyfrowane przy użyciu algorytmu Bcrypt.
  • Walidacja danych wejściowych za pomocą modeli Pydantic.

Dostępne Endpointy

  • POST /register — Rejestracja nowego użytkownika
  • POST /login — Logowanie i uzyskanie tokena dostępu
  • GET /products — Pobranie listy wszystkich produktów
  • POST /products — Dodanie nowego produktu do bazy
  • POST /orders — Złożenie nowego zamówienia (wymaga tokena)
